# Pack-to-Curriculum Map

This document turns the current content pack library into a curriculum-ready sequence.

It does **not** replace adaptive logic. Instead, it gives the adaptive engine a structured backbone:

- grade-band roadmap
- curriculum units
- ordered pack sequence
- core vs support placement
- suggested session pacing
- review points
- progression logic

---

# 1. How to Use This Map

## Core principle
Each grade band should have:
- a **planned core sequence**
- **adaptive reinforcement** layered on top
- **review checkpoints** between unit clusters

## Recommended session model
For each student:
- **Core sessions** come from the mapped pack sequence
- **Adaptive sessions** pull weak-skill activities from completed/current packs
- **Review sessions** revisit earlier packs and skills
- **Bonus sessions** can be generated from optional packs later

## Recommended pacing
A practical starting pacing model:
- **2 core sessions per week**
- **1 adaptive/review session per week** when needed

This keeps curriculum movement visible while still allowing mastery support.

# 5. Core vs Review vs Adaptive Rules

## Core packs
All packs listed in the sequences above should be marked:
- `is_core = true`
- `phase = core_cycle_1`

## Review checkpoints
At each checkpoint, generate:
- 1 review session from the previous 2–3 packs
- 1 adaptive session based on weakest skills seen in those packs

## Adaptive insertion rule
If mastery is below threshold in a unit’s primary skill:
- keep the student in the current unit cluster longer
- insert remediation from already completed packs in the same cluster
- do not immediately jump to unrelated future packs

That prevents the app from turning back into a random weak-skill chase.

# 7. Recommended Backend Use

## Session queue logic
For each student:
1. assign grade band
2. assign curriculum sequence for that band
3. unlock the first core pack
4. generate sessions from that pack until completion
5. mark checkpoint after unit cluster
6. inject adaptive/review if needed
7. unlock next pack

## Pack completion rule
A pack can be considered complete when:
- required core activities from the pack have been served
- minimum evidence threshold is met for primary skills
- student has either reached target range or received at least one adaptive follow-up

## Important
The adaptive engine should choose from:
- current pack
- previous related packs in same unit
- current checkpoint review pool

It should **not** jump freely across the entire grade band unless explicitly in bonus mode.
